## 🎯 插件简介

**Moka简历筛选Agent** 是一款专为招聘场景设计的Chrome浏览器扩展，能够：

- 🤖 **自动筛选**：基于大模型智能评估候选人简历
- 📄 **批量处理**：支持分页和无限滚动两种页面模式
- 📊 **结果导出**：一键导出CSV格式的筛选结果
- 🔄 **自动推荐**：符合条件的候选人可自动推荐给用人部门
- ⏸️ **灵活控制**：支持暂停、继续、停止等操作

### 适用场景

- 需要在Moka招聘系统中批量筛选候选人
- 需要根据学历、专业、经验等条件快速初筛简历
- 需要将筛选结果导出为Excel/CSV格式

## 📦 安装步骤

### 方式一：开发者模式安装（推荐）

1. **下载插件文件**
   - 将插件文件夹保存到本地电脑

2. **打开Chrome扩展管理页面**
   - 在Chrome地址栏输入：`chrome://extensions/`
   - 或者点击菜单 → 更多工具 → 扩展程序

3. **开启开发者模式**
   - 在页面右上角找到「开发者模式」开关
   - 将开关切换到**开启**状态

4. **加载插件**
   - 点击左上角的「加载未打包的扩展程序」按钮
   - 选择插件文件夹（包含 `manifest.json` 的文件夹）
   - 插件图标将出现在浏览器工具栏

### 方式二：验证安装成功

- 看到浏览器工具栏出现插件图标 ✅
- 点击图标能打开侧边栏面板 ✅

## 🚀 使用流程

### 第一步：准备工作

1. **打开Moka招聘系统**
   - 登录Moka系统
   - 进入候选人列表页面（如：人才库、简历筛选页面）

2. **打开插件面板**
   - 点击浏览器工具栏的插件图标
   - 或使用快捷键（如有设置）

### 第二步：配置大模型

1. 在「大模型配置」区域填写：
   - **模型类型**：如 `doubao-seed-1-6-lite`、`gpt-4o` 等
   - **API Key**：从对应平台获取的API密钥
   - **Base URL**：API接口地址

2. 点击「保存配置」按钮

> 🔐 **安全提示**：API Key仅保存在本地浏览器中，不会上传到任何服务器

### 第三步：设置筛选条件

1. **初筛标准**（简单条件，用于快速过滤）
   ```
   示例：
   - 985院校，计算机相关专业
   - 本科及以上学历，2024年及以后毕业
   ```

2. **完整筛选标准**（详细条件，用于深度评估）
   ```
   示例：
   - 211或者QS前100院校
   - 有算法相关经验或项目经历
   - 熟悉Python/Java等编程语言
   ```

3. **设置筛选数量**
   - 输入计划筛选的简历总数
   - 系统会自动翻页或滚动加载，直到达到目标数量

4. **自动推荐**（可选）
   - 勾选「直接推荐合适人选给用人团队」
   - 填写负责人姓名（多个用逗号分隔）

### 第四步：开始筛选

1. 点击「开始筛选」按钮
2. 观察实时日志，查看处理进度
3. 系统会自动：
   - 提取候选人列表信息
   - 使用大模型进行初筛
   - 进入候选人详情页获取简历
   - 对简历进行详细评估
   - 保存筛选结果

### 第五步：导出结果

1. 筛选完成后，点击「导出结果 CSV」按钮
2. 系统会自动下载CSV文件：
   - `合格候选人_YYYY-MM-DD.csv` - 符合筛选条件的候选人
   - `API失败候选人_YYYY-MM-DD.csv` - 评估过程中出错的候选人（如有）

### CSV文件格式

| 字段 | 说明 |
|------|------|
| 姓名 | 候选人姓名 |
| 手机号 | 候选人电话 |
| 邮箱 | 候选人邮箱 |
| 详情页URL | 候选人详情页链接 |
| 通过筛选原因 | AI评估的具体理由 |
| 最高学历院校 | 最高学历学校名称 |
| 最高学历专业 | 最高学历专业名称 |
| 本科院校 | 本科学校名称 |
| 本科专业 | 本科专业名称 |
| 备注 | 特殊情况说明（如：已被锁定） |

## ⚙️ 配置说明

### 筛选标准编写技巧

**初筛标准**：
- 简洁明了，用于快速过滤
- 关注硬性条件（学历、学校、专业、毕业时间）
- 示例：`985/211院校，计算机相关专业，本科及以上学历，26年毕业`

**完整筛选标准**：
- 详细具体，用于深度评估
- 可包含技能要求、经验要求、软性条件
- 示例：
  ```
  1. 学历：本科及以上学历
  2. 学校：985/211院校或海外知名院校
  3. 专业：计算机、软件工程、人工智能等相关专业
  4. 技能：熟悉Python/Java，有机器学习项目经验
  5. 经验：有互联网大厂实习经历优先
  ```

**简历筛选负责人**：
- 填写花名，必须确保准确
- 需要多位业务同学筛选的话可以填写多个花名，用逗号隔开。比如：`曜霖,张三,李四`

## ❓ 常见问题

### Q1: 插件无法打开/不显示侧边栏？

**解决方法**：
1. 确认已正确安装插件（开发者模式开启并加载）
2. 刷新Moka页面后重试
3. 检查浏览器控制台是否有错误信息

### Q2: API Key 填错了怎么办？

**解决方法**：
- 直接在「API Key」输入框修改正确的Key
- 点击「保存配置」按钮
- 无需重新安装插件

### Q3: 筛选过程中可以关闭浏览器吗？

**回答**：
- ❌ 不建议关闭浏览器，筛选会中断
- ✅ 可以切换到其他标签页，筛选会在后台继续
- ✅ 可以使用「暂停」功能暂停筛选，稍后「继续」

### Q4: 如何暂停和继续筛选？

**操作步骤**：
1. 筛选过程中点击「暂停」按钮
2. 系统会暂停在当前候选人
3. 需要继续时点击「继续」按钮
4. 系统会从暂停处恢复筛选

### Q5: 导出CSV文件在哪里？

**查找方法**：
- 文件会自动下载到浏览器的默认下载文件夹
- 文件名格式：`合格候选人_2024-01-15.csv`
- 可在Chrome下载历史中查看（Ctrl+J）

### Q6: 为什么有些候选人显示"API失败"？

**可能原因**：
- 网络连接不稳定
- API Key 失效或额度不足
- 大模型服务暂时不可用

**解决方法**：
- 检查网络连接
- 确认API Key有效且有足够额度
- 稍后重新筛选失败的候选人

### Q7: 自动推荐功能不工作？

**检查事项**：
1. 确认已勾选「直接推荐合适人选给用人团队」
2. 确认「简历筛选负责人」已填写正确姓名
3. 确认当前页面类型支持推荐功能（无限滚动页面支持，分页页面暂不支持）
4. 检查候选人是否已被其他岗位锁定

### Q8: 如何修改筛选条件重新筛选？

**操作步骤**：
1. 点击「停止」按钮结束当前筛选
2. 修改「初筛标准」和「完整筛选标准」
3. 点击「开始筛选」重新启动
