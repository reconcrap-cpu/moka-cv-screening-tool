### 详细步骤

#### 步骤1: 开发完成
- Backend Agent完成功能开发
- 运行本地测试确保功能正常
- 准备提交文件列表

- 编写符合规范的提交消息

#### 步骤2: 发送提交请求
- Backend Agent向Git Controller Agent发送提交请求
- 提供完整的提交信息:
  - 模块名称
  - 文件列表
  - 提交消息
  - 分支名称

#### 步骤3: 执行Git操作
- Git Controller Agent验证请求信息
- 检查当前分支状态
- 执行git add命令暂存文件
- 执行git commit命令提交代码
- 执行git push命令推送到远程

- 记录操作日志

#### 步骤4: 创建Pull Request
- Git Controller Agent创建Pull Request
- 设置PR标题和描述
- 指定审查者
- 设置目标分支为develop
- 记录PR编号

#### 步骤5: 代码审查
- Architect Agent评审代码架构
- Test Agent检查测试覆盖率
- 审查结果记录在PR中
- 通过则Approve，- 不通过则Request Changes

#### 步骤6: 合并Pull Request
- Git Controller Agent检查PR状态
- 所有审查者Approve后执行合并
- 使用Squash merge方式合并
- 删除功能分支
- 通知相关Agent合并完成

## 冲突解决流程

### 流程图

```
┌─────────────────────────────────────────────────────────────┐
│  Git Controller Agent: 检测到合并冲突                         │
│  - 冲突文件: src/modules/candidate/CandidateExtractor.js     │
│  - 冲突原因: Backend Agent A和Backend Agent B同时修改         │
└─────────────────────┬───────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────────┐
│  Git Controller Agent: 分析冲突                               │
│  1. 提取冲突内容                                              │
│  2. 识别冲突类型: 内容冲突/结构冲突                            │
│  3. 评估冲突影响范围                                          │
└─────────────────────┬───────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────────┐
│  Git Controller Agent: 通知相关Agent                          │
│  - 通知Backend Agent A: 你的修改与Agent B冲突                 │
│  - 通知Backend Agent B: 你的修改与Agent A冲突                 │
│  - 通知Architect Agent: 需要决策冲突解决方案                  │
└─────────────────────┬───────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────────┐
│  Architect Agent: 制定冲突解决方案                            │
│  - 分析两方修改的合理性                                        │
│  - 决定保留哪部分代码或合并两者                                │
│  - 将解决方案发送给Git Controller Agent                       │
└─────────────────────┬───────────────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────────────┐
│  Git Controller Agent: 执行冲突解决                           │
│  1. 应用Architect Agent的解决方案                             │
│  2. 验证解决后的代码无冲突                                     │
│  3. 提交解决后的代码                                          │
│  4. 继续合并流程                                              │
└─────────────────────────────────────────────────────────────┘
```

### 冲突类型

| 冲突类型 | 描述 | 解决方式 |
|---------|------|---------|
| 内容冲突 | 同一文件的不同部分被修改 | 自动合并或保留最新修改 |
| 结构冲突 | 同一文件的相同部分被修改 | Architect Agent决策 |
| 接口冲突 | 接口定义被修改 | Architect Agent决策，通知所有依赖方 |

### 冲突预防措施
1. **模块文件隔离**: 每个模块独立目录，禁止跨模块修改文件
2. **接口先行**: 先定义接口，后实现功能
3. **分支隔离**: 每个Agent在独立分支开发
4. **定期同步**: 每日从develop分支同步更新

## 分支管理规范

### 分支类型
| 分支类型 | 命名格式 | 用途 | 生命周期 |
|---------|---------|------|---------|
| 主分支 | main | 生产环境代码 | 永久 |
| 开发分支 | develop | 开发环境代码 | 永久 |
| 功能分支 | feature/* | 功能开发 | 临时 |
| 修复分支 | bugfix/* | 缺陷修复 | 临时 |
| 热修复分支 | hotfix/* | 紧急修复 | 临时 |
| 发布分支 | release/* | 版本发布 | 临时 |

### 分支命名规范
- **功能分支**: feature/模块名称
  - 示例: feature/candidate-extractor
- **修复分支**: bugfix/问题描述
  - 示例: bugfix/selector-not-found
- **热修复分支**: hotfix/问题描述
  - 示例: hotfix/api-rate-limit
- **发布分支**: release/版本号
  - 示例: release/v3.0.0

### 分支保护规则
- **main分支**: 禁止直接提交，必须通过PR合并
- **develop分支**: 禁止直接提交，必须通过PR合并
- **功能分支**: 可以直接提交，但需要经过审查才能合并

## 提交规范

### 提交消息格式
- **格式**: 类型(范围): 描述
- **类型**: feat, fix, refactor, docs, test, chore
- **范围**: 模块名称或影响范围
- **示例**:
  - feat(candidate): 实现候选人信息提取功能
  - fix(selector): 修复选择器失效问题
  - refactor(utils): 重构工具函数
  - docs(readme): 更新README文档
  - test(candidate): 添加候选人提取测试
  - chore(deps): 更新依赖版本

### 提交检查清单
- [ ] 代码符合项目规范
- [ ] 无硬编码配置
- [ ] 无敏感信息泄露
- [ ] 测试覆盖充分
- [ ] 文档已同步更新

## 附录

### A. Git命令速查

| 命令 | 用途 | 示例 |
|------|------|------|
| git status | 查看状态 | git status |
| git add | 暂存文件 | git add . |
| git commit | 提交代码 | git commit -m "message" |
| git push | 推送代码 | git push origin branch |
| git pull | 拉取代码 | git pull origin branch |
| git merge | 合并分支 | git merge branch |
| git branch | 管理分支 | git branch -a |
| git checkout | 切换分支 | git checkout branch |
| git tag | 管理标签 | git tag -a v1.0.0 |
| git log | 查看日志 | git log --oneline |

### B. 常见问题解决

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 推送失败 | 网络问题 | 重试或检查网络 |
| 合并冲突 | 代码冲突 | 按照冲突解决流程处理 |
| 分支不存在 | 分支未创建 | 先创建分支 |
| 权限不足 | Token权限不够 | 检查Token权限配置 |
