# 人选筛选助手 Chrome Extension 使用说明

---

## 一、插件简介

**人选筛选助手**是一款专为招聘场景设计的 Chrome 浏览器插件，能够自动从 Moka 招聘系统中提取候选人信息，并通过大语言模型进行智能筛选，大幅提高简历筛选效率。

### 主要功能

- 🤖 **AI 智能筛选**：使用大语言模型自动分析候选人简历
- 📄 **简历自动抓取**：自动提取候选人详情页简历图片
- 📊 **批量处理**：支持分页和无限滚动两种页面模式
- 📥 **结果导出**：一键导出符合条件的候选人 CSV 文件
- 🔄 **自动推荐**：可将合适人选自动推荐给用人团队

---

## 二、安装步骤

### 步骤 1：下载插件文件

确保你拥有以下文件：
- `manifest.json`
- `content.js`
- `sidebar.html`
- `sidebar.js`
- `background.js`
- `config.js`
- `html2canvas.min.js`
- `images/` 文件夹（包含图标文件）

### 步骤 2：在 Chrome 中加载插件

1. 打开 Chrome 浏览器，在地址栏输入：`chrome://extensions/`
2. 打开右上角的**开发者模式**开关
3. 点击左上角的**加载已解压的扩展程序**按钮
4. 选择包含插件文件的文件夹
5. 插件安装成功后，会在扩展程序列表中显示"Moka简历筛选Agent"

### 步骤 3：固定插件图标

1. 点击 Chrome 工具栏的**扩展程序**图标（拼图形状）
2. 找到"Moka简历筛选Agent"
3. 点击**固定**按钮，将插件图标固定到工具栏

## 四、配置说明

### 4.1 大模型配置

首次使用需要配置大模型 API 信息：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| **模型类型** | 使用的大模型名称 | `doubao-seed-1-6-lite` |
| **API Key** | 你的 API 密钥 | 从模型服务商获取 |
| **Base URL** | API 请求地址 | `https://ark.cn-beijing.volces.com/api/v3` |

**支持的模型服务商：**
- 火山引擎（豆包大模型）
- OpenAI
- 其他兼容 OpenAI API 格式的服务商

**配置步骤：**
1. 在对应区域输入模型类型、API Key 和 Base URL
2. 点击**保存配置**按钮
3. 配置会自动保存，下次打开无需重新输入

### 4.2 筛选配置

#### 初筛标准

用于第一轮快速筛选，判断候选人是否值得进一步查看简历。

**示例：**
- `985院校，理工专业`
- `本科及以上学历，计算机相关专业`
- `3年以上工作经验`

#### 完整筛选标准

用于详细评估候选人简历，做出最终筛选决定。

**示例：**
- `211或者QS前100院校，并且有算法相关经验`
- `有互联网大厂工作经验，熟悉Java/Python`
- `硕士学历，发表过顶会论文`

#### 目标数量

设置需要筛选的简历总数，插件会自动翻页直到达到目标数量或没有更多简历。

#### 自动推荐功能

勾选**直接推荐合适人选给用人团队**后：
- 符合条件的候选人会自动推荐给指定负责人
- 需要在下方输入负责人姓名（多个用逗号分隔）

## 五、使用流程

### 5.1 准备工作

1. **登录 Moka 系统**：在浏览器中打开 Moka 招聘系统并登录
2. **进入人才库**：导航到需要筛选的候选人列表页面
3. **打开插件**：点击 Chrome 工具栏的插件图标

### 5.2 开始筛选

1. **检查配置**：确保大模型配置和筛选配置已正确填写
2. **点击开始**：点击**开始筛选**按钮
3. **等待处理**：插件会自动：
   - 提取当前页面候选人信息
   - 使用 AI 进行初筛
   - 对通过初筛的候选人进入详情页
   - 抓取简历图片并进行详细评估
   - 自动翻页继续处理

### 5.3 监控进度

- 在**状态栏**查看当前进度
- 在**实时日志**区域查看详细处理信息
- 可以随时点击**暂停**按钮暂停筛选

### 5.4 导出结果

筛选完成后：
1. 点击**导出结果 CSV**按钮
2. 系统会自动下载两个文件（如有数据）：
   - `合格候选人_YYYY-MM-DD.csv` - 符合筛选条件的候选人
   - `API失败候选人_YYYY-MM-DD.csv` - API 调用失败的候选人

### 5.5 CSV 文件格式

导出的 CSV 文件包含以下字段：

| 字段 | 说明 |
|------|------|
| 姓名 | 候选人姓名 |
| 手机号 | 候选人手机号 |
| 邮箱 | 候选人邮箱 |
| 详情页URL | 候选人详情页链接 |
| 通过筛选原因 | AI 评估的通过理由 |
| 最高学历院校 | 最高学历毕业院校 |
| 最高学历专业 | 最高学历专业 |
| 本科院校 | 本科毕业院校 |
| 本科专业 | 本科专业 |
| 备注 | 特殊状态说明 |

## 七、常见问题

### Q1: 插件无法启动怎么办？

**解决方法：**
1. 检查是否已正确安装插件（开发者模式是否开启）
2. 刷新 Moka 页面后重试
3. 检查 Chrome 控制台是否有错误信息

### Q2: API 调用失败怎么办？

**可能原因及解决：**
- **API Key 错误**：检查 API Key 是否正确
- **网络问题**：检查网络连接
- **模型名称错误**：确认模型类型名称正确
- **余额不足**：检查 API 账户余额

### Q3: 无法获取简历图片？

**解决方法：**
1. 确保候选人详情页已完全加载
2. 检查简历是否正常显示
3. 尝试刷新页面后重新开始

### Q4: 筛选结果不准确？

**优化建议：**
1. **优化筛选标准**：使用更具体、明确的描述
2. **调整模型参数**：尝试不同的模型
3. **分批筛选**：减少单次筛选数量，提高准确性

### Q5: 自动推荐功能无效？

**检查事项：**
1. 确认已勾选"直接推荐合适人选给用人团队"
2. 确认已填写负责人姓名
3. 确认当前页面支持推荐功能（无限滚动页面）
4. 检查候选人是否已被其他岗位锁定

## 八、注意事项

1. **页面保持**：筛选过程中请勿关闭或刷新 Moka 页面
2. **网络稳定**：确保网络连接稳定，避免 API 调用失败
3. **API 限额**：注意 API 调用次数限制和费用
4. **隐私保护**：妥善保管 API Key，不要分享给他人
5. **结果复核**：AI 筛选结果仅供参考，建议人工复核重要岗位
