## 三、简历格式多样性测试

### 3.1 发现的简历格式

通过500人测试，发现以下简历存储格式：

| 格式 | 选择器 | 说明 |
|------|--------|------|
| **img-page** | `.img-page` | 主要格式，PDF转换的图片 |
| **resume-img** | `.resume-XXXXX` | **新发现**：动态类名格式 |
| **iframe** | `#iframeResume` | iframe嵌入格式，含SVG子格式 |
| **img-blob** | `.img-blob` | Blob URL格式（代码已支持） |

### 3.2 关键发现

**✅ 所有候选人都有简历！**

之前报告的"79.5%无简历"是因为检测代码遗漏了新格式 `.resume-XXXXX`（动态类名）。

**问题根因**：
- 原代码只检测 `.img-page` 类
- 部分简历使用动态类名如 `.resume-2IjJ_`
- 这类图片同样存储在 OSS 上，格式为 `.jpeg` 或 `.jpg`

### 3.3 代码修复

已更新 `ResumeImageModule.js` 的 `tryPageImages` 函数：

```javascript
// 支持多种简历图片格式：
// 1. .img-page - 主要格式
// 2. .resume-XXXXX - 动态类名格式（如 .resume-2IjJ_）
// 3. img[src*="pdf_"] - PDF转换的图片

// 查找所有带 resume- 前缀类名的图片
var allImages = document.querySelectorAll('img');
var resumeImages = [];
for (var i = 0; i < allImages.length; i++) {
  var img = allImages[i];
  if (img.className && (
    img.className.indexOf('resume-') === 0 ||
    img.className.indexOf(' resume-') !== -1
  )) {
    resumeImages.push(img);
  }
}
```
